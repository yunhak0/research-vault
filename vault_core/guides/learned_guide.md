# Learned 문서 작성 가이드

## 1. 목적

**실패한 시도에서 얻은 교훈**을 기록하는 문서.
"이것은 왜 작동하지 않았는가?"에 답한다.
같은 실수를 반복하지 않기 위한 지식 자산.

## 2. ID 규칙

```
LRN-[NNN]
```
LRN 번호는 PRB 내에서만 유일하면 된다.

## 3. 파일 위치

```
[PROJECT]/problems/PRB-[NNN]_xxx/learned/LRN-[NNN]_[name].md
```

## 4. LRN 문서 생성 조건

- INV에서 시도한 접근이 **실패**했을 때
- 아이디어가 검토 결과 **기각**되었을 때
- 실험 결과 가설이 **기각**되었을 때

**핵심**: 단순한 "실패 기록"이 아니라 **교훈 추출**이 목적.

## 5. 섹션별 작성 규칙

### 5.1 교훈 — `> **교훈**: ...`로 시작하는 한 문장. 핵심 takeaway.
### 5.2 시도한 접근 — 배경 및 동기 + 구체적 방법
### 5.3 실험 결과 — 정량적 결과 테이블 필수. W&B Run ID 참조 권장.
### 5.4 실패 원인 분석 — 표면적 원인 + 근본 원인. LRN의 핵심 가치.
### 5.5 교훈 (상세) — 번호 매긴 교훈 목록. 일반화 가능한 형태로.
### 5.6 이 시도가 배제하는 것 — 더 이상 시도할 필요 없는 접근들.

## 6. Status 정의

| 상태 | 정의 |
|------|------|
| Rejected | 접근이 기각됨 (일반적) |
| Completed | 교훈 정리 완료 (기각이 아닌 학습 완료) |

## 7. LRN vs RES 판단 기준

| 조건 | 결과 |
|------|------|
| 시도가 문제를 악화시킴 | LRN |
| 시도가 효과 없음 | LRN |
| 시도가 부분적 성공, 채택 안 함 | LRN |
| 시도가 문제를 해결함 | RES |
| 시도가 부분적 성공, 채택함 | RES |

## 8. Notion 아카이빙

LRN은 개별 아카이빙하지 않는다. 핵심 교훈은 _overview.md를 통해 아카이빙됨.
