# Daily Note 작성 가이드

## 1. 목적

매일의 연구 활동을 기록하여:
- 진행 상황 추적
- 아이디어 및 메모 임시 저장
- 업무 연속성 유지
- 회고를 통한 개선

## 2. ID 규칙

```
DAILY-[YYYY-MM-DD]

예시: DAILY-2025-02-03
```

## 3. 파일 위치

```
01_Global/daily/DAILY-[YYYY-MM-DD].md

예시: 01_Global/daily/DAILY-2026-02-03.md
```

## 4. 섹션별 작성 규칙

### 4.1 오늘의 목표 (Today's Goals)

**목적**: 오늘 달성하려는 구체적 목표 설정

**작성 규칙**:
- 2-5개 목표 권장
- 완료 여부 판단 가능한 형태로
- 프로젝트/실험 ID 포함
- 우선순위 순서로 나열

**형식**: 체크리스트

**좋은 예**:
```
- [ ] PRB-002 INV-001 Phase 0 Oracle bound 계산 완료
- [ ] LIT-013 문헌 리뷰 작성 (FlowMatching 논문)
- [ ] 주간 미팅 자료 준비
```

**나쁜 예**:
```
- [ ] 연구하기
- [ ] 논문 읽기
```
(너무 모호, 완료 판단 불가)

### 4.2 작업 기록 (Work Log)

**목적**: 실제 수행한 작업 기록

**작성 규칙**:
- 프로젝트별로 구분
- 수행/문제/해결 구조
- 관련 문서 링크 포함
- 시간 기록 (선택)

**형식**:

```markdown
### P001_scDiffuser

**수행:**
- [09:00-12:00] PRB-002 Spearman gap 진단 실험 실행
- [14:00-16:00] Oracle bound 계산 결과 확인

**문제:**
- Spearman 계산 시 ties 처리 방식에 따라 결과 달라짐

**해결:**
- scipy.stats.spearmanr 기본 설정 (average ties) 사용으로 통일

**관련 문서:**
- [[INV-001_diagnostic_first_plan]]
```

### 4.3 아이디어/메모 (Ideas & Notes)

**목적**: 작업 중 떠오른 생각 임시 저장

**작성 규칙**:
- 자유 형식
- 중요한 아이디어는 별도 아이디어 노트로 발전
- 나중에 확인할 것 표시

### 4.4 내일 할 일 (Tomorrow's Tasks)

**목적**: 다음 날 업무 연속성 확보

**작성 규칙**:
- 오늘 작업 기반으로 작성
- 우선순위 순서
- 구체적으로

**형식**: 체크리스트

### 4.5 회고 (Retrospective)

**목적**: 하루를 돌아보며 개선점 발견

**작성 규칙**:
- 선택사항이지만 권장
- 솔직하게 작성
- 개선 가능한 부분 중심

**형식**: 잘된 점 / 개선할 점 / 배운 점

## 5. Daily Note 생성 타이밍

**권장 패턴**:
1. **아침**: 오늘의 목표 작성
2. **작업 중**: 작업 기록 실시간 업데이트
3. **퇴근 전**: 내일 할 일 + 회고 작성

## 6. Daily Note와 다른 문서의 관계

| Daily Note 내용 | 발전 시 | 대상 문서 |
|----------------|--------|----------|
| 아이디어 | 구체화 가치 있음 | INV 문서 (investigating) |
| 실험 진행 기록 | 상세 기록 필요 | INV 또는 RES 문서 |
| 문헌 메모 | 정식 정리 필요 | LIT 문헌 리뷰 |
| 새 문제 발견 | 문제 정의 필요 | PRB 폴더 + _overview.md |
| 회고의 개선점 | 패턴 발견 | FND 또는 LRN 문서 |

## 7. Obsidian ↔ Notion 동기화 규칙

- **동기화하지 않음** (Daily Note는 working 문서)
- 예외: 중요한 마일스톤 날짜만 선택적으로

## 8. 흔한 실수와 주의사항

| 실수 | 문제점 | 올바른 방법 |
|------|--------|------------|
| 목표 없이 시작 | 방향 없는 작업 | 아침에 목표 먼저 설정 |
| 작업 후 한꺼번에 기록 | 세부 사항 누락 | 실시간 또는 자주 기록 |
| 문제만 기록, 해결 누락 | 같은 문제 반복 | 해결 방법 반드시 기록 |
| 링크 없는 기록 | 맥락 추적 어려움 | 관련 문서 링크 포함 |
| 회고 생략 | 개선 기회 놓침 | 짧게라도 회고 작성 |
